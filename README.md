## How I Think

I approach every build as a systems problem first.

Before writing a line of code, I ask: what's the data flow, where are the failure points, and what does success actually look like for the person using this? I'm not interested in building things that only work in demos. I optimize for real-world robustness — graceful error handling, sensible defaults, systems that degrade well under pressure.

Speed matters, but so does judgment. I move fast by making better decisions earlier, not by skipping them.
